    Documentação da View complete_login

    Visão Geral

    A view complete_login é um componente essencial no processo de autenticação de um sistema multi-tenant. Ela representa a segunda etapa de um login em duas fases: após o usuário ser identificado, essa view recebe a senha, valida a existência da empresa (tenant) via subdomínio e autentica as credenciais do usuário no schema correto daquela empresa.

    Modelos e Tabelas Utilizados

    • Client (Tenant): Modelo principal do django-tenants, representa uma empresa/cliente. Serve para identificar o schema (banco de dados) com base no subdomínio acessado.

    • User (Usuário): Modelo de usuário do Django (padrão ou customizado). A autenticação verifica as credenciais dentro do schema do tenant ativo.

    Fluxo de Execução da View

    1. Requisição GET — Exibição da Página

    Objetivo: Renderizar a página para o usuário digitar sua senha.

    Processo:

    1. O usuário é redirecionado para essa view após a primeira etapa de login (identificação do username).
    2. A função renderiza o template client_app/auth/complete_login.html.
    3. O username é passado no contexto do template (ex: "Olá, fulano. Por favor, digite sua senha.").

    2. Requisição POST — Submissão do Formulário

    Objetivo: Validar a senha, o tenant e efetuar o login.

    Processo:

    1. Coleta de Dados

      • A senha é extraída do request.POST.

    2. Identificação do Tenant

      • O host da requisição (ex: empresa-a.seusite.com) é capturado.
      • O subdomínio (ex: empresa-a) é isolado e usado como schema_name.
      • O sistema tenta buscar um objeto Client correspondente a esse schema_name.

    3. Validação do Tenant

      • Se não for encontrado, exibe-se uma mensagem de erro ("Empresa inexistente") e o usuário é redirecionado para a página de login pública.

    4. Ativação do Schema do Tenant

      • Se o tenant existir, o sistema troca o schema de banco de dados para o da empresa com schema_context(). A partir daí, todas as operações de banco de dados ocorrerão somente dentro daquele tenant.

    5. Autenticação do Usuário

      • A função authenticate() é chamada com o username e a password.

      • Se a autenticação for bem-sucedida:

        • A função login() é chamada para iniciar a sessão.
        • O usuário é redirecionado para a página principal da empresa (client_app:client_home).

      • Se falhar:

        • Uma mensagem de erro ("Usuário ou senha incorretos") é exibida.
        • O usuário é redirecionado para a página de login pública.

    Dependências e Contexto

    • django-tenants: Biblioteca que permite multi-tenancy no Django. A identificação do schema e uso do schema_context() são cruciais para garantir a separação entre empresas.

    • Autenticação Django: Usa authenticate() e login() do módulo django.contrib.auth.

    • Framework de Mensagens: Usa messages.error() para exibir mensagens de erro sem interromper o fluxo.

    Considerações Finais

    A função complete_login é uma implementação robusta e segura para o login em ambientes multi-tenant. Ela garante a separação lógica e de dados entre os diferentes clientes da plataforma, direcionando corretamente a tentativa de autenticação para o banco de dados isolado da empresa correspondente, o que é um pilar fundamental de segurança e funcionamento para este tipo de arquitetura.
